================================================================================
REACT BASICS — HOW IT WORKS
================================================================================

--------------------------------------------------------------------------------
1. WHAT IS REACT?
--------------------------------------------------------------------------------

Technical definition:
  React is a JavaScript library developed by Facebook/Meta for building user
  interfaces. It is declarative, component-based, and handles only the View
  layer of an application.

Simple way:
  React is a tool that helps you build the visual part of a website (what you
  see and click on). You describe WHAT the page should look like, and React
  figures out HOW to show it in the browser.

--------------------------------------------------------------------------------
2. HOW REACT WORKS — CORE CONCEPTS
--------------------------------------------------------------------------------

--- VIRTUAL DOM ---

Technical definition:
  The Virtual DOM (VDOM) is a lightweight in-memory representation of the
  actual browser DOM (Document Object Model). When state or props change, React
  re-renders the Virtual DOM, performs a "diffing" algorithm to find the minimum
  number of changes, and then applies only those changes to the real DOM.
  This process is called "reconciliation."

Simple way:
  The real browser DOM is slow to update. So React keeps a copy of the page
  layout in memory (the virtual DOM). When something changes, React first
  updates its memory copy, compares the old and new versions, and then only
  updates the parts of the real page that actually changed — instead of
  redrawing everything.

Example:
  If you have a list of 100 items and only item #3 changes, React updates only
  that one item in the browser, not all 100.


--- COMPONENTS ---

Technical definition:
  A component is a reusable, self-contained unit that returns JSX (a description
  of UI). Components can be functions or classes. They accept inputs called
  "props" and can maintain internal state.

Simple way:
  A component is like a LEGO brick. You build small pieces (a button, a navbar,
  a card) and assemble them together to build a full page. Each piece manages
  its own appearance.

Example:
  A <Button> component, a <Header> component, and a <UserCard> component are
  all separate components that you combine to build a page.


--- JSX (JavaScript XML) ---

Technical definition:
  JSX is a syntax extension for JavaScript that allows you to write HTML-like
  markup inside JavaScript files. It is not valid JavaScript by default — it
  must be transpiled (converted) into regular JavaScript function calls by a
  tool like Babel or the Vite React plugin before the browser can understand it.

Simple way:
  JSX lets you write something that looks like HTML inside your JavaScript file.
  The browser cannot read JSX directly, so a tool converts it into plain
  JavaScript before sending it to the browser.

Example:
  You write:    <h1>Hello World</h1>
  Converted to: React.createElement("h1", null, "Hello World")


--- STATE ---

Technical definition:
  State is a mutable data structure managed internally by a component. When
  state changes, React triggers a re-render of that component and its children.

Simple way:
  State is a component's memory. If a button has been clicked 3 times, the
  component "remembers" that number. When the number changes, React automatically
  updates what is shown on screen.


--- PROPS ---

Technical definition:
  Props (short for properties) are read-only inputs passed from a parent
  component to a child component. They allow data to flow downward in the
  component tree.

Simple way:
  Props are like arguments you pass to a function. A parent component sends
  information to a child component through props, and the child uses that
  information to decide what to display.

--------------------------------------------------------------------------------
3. WHAT REACT NEEDS TO RUN
--------------------------------------------------------------------------------

React does not run on its own. It needs several tools and packages.

--- Node.js ---

Technical definition:
  Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows
  JavaScript to run outside of the browser, on your local machine. It is
  required to use npm and to run build tools.

Simple way:
  Normally JavaScript only runs inside a browser. Node.js lets JavaScript run
  on your computer directly, so tools like React's build system can operate.

--- npm (Node Package Manager) ---

Technical definition:
  npm is the default package manager for Node.js. It resolves, downloads, and
  manages third-party JavaScript packages listed in package.json.

Simple way:
  npm is like an app store for JavaScript code. When your project needs a
  library, npm downloads it and stores it in the node_modules folder.

--- node_modules folder ---

Technical definition:
  A directory created by npm that contains all installed dependencies and their
  transitive dependencies (dependencies of dependencies).

Simple way:
  This is where npm stores every library your project uses. This folder can
  contain hundreds of sub-folders. You never edit this folder manually.

--- package.json ---

Technical definition:
  A JSON manifest file that declares the project's name, version, scripts,
  and dependency list (both runtime and development dependencies).

Simple way:
  This file is the project's ID card. It lists every library the project needs
  and the commands you can run (like npm run dev).

--- react (core library) ---

Technical definition:
  The core React library. It provides the component model, state management
  hooks, the Virtual DOM diffing logic, and the JSX factory function
  (React.createElement).

Simple way:
  This is React itself. It contains the logic that makes components, state,
  and the virtual DOM work.

--- react-dom ---

Technical definition:
  A separate package that serves as the bridge between React's Virtual DOM and
  the browser's actual DOM. It provides ReactDOM.createRoot() which mounts the
  React application into a real HTML element.

Simple way:
  React does not know anything about the browser by itself. react-dom is the
  part that takes React's output and actually puts it onto the real webpage.
  React and react-dom are always installed together.

--- Bundler (e.g., Vite or Webpack) ---

Technical definition:
  A bundler is a build tool that takes your source files (JSX, CSS, images,
  modules), resolves all import/export statements, transforms JSX into
  JavaScript, and outputs optimized files the browser can load.

  - Vite: A modern bundler that uses native ES Modules in development and
    Rollup for production builds. Much faster than Webpack.
  - Webpack: An older, more configurable bundler. Used by Create React App.

Simple way:
  Your project is split across many files. The bundler glues them all together
  into files the browser can actually understand and load. It also converts JSX
  into plain JavaScript.

--- Babel / @vitejs/plugin-react ---

Technical definition:
  Babel is a JavaScript transpiler (source-to-source compiler) that converts
  modern JavaScript and JSX syntax into backwards-compatible JavaScript.
  When using Vite, the @vitejs/plugin-react plugin handles JSX transformation
  using either Babel or the SWC compiler internally.

Simple way:
  JSX and some modern JavaScript features are not understood by older browsers.
  These tools convert your code into a version that all browsers can read.

--------------------------------------------------------------------------------
4. WHAT HAPPENS WHEN YOU RUN "npm run dev"
--------------------------------------------------------------------------------

"npm run dev" starts a DEVELOPMENT SERVER. It does NOT produce a final build.

--- Development Server ---

Technical definition:
  A local HTTP server (typically started by Vite) that serves your application
  files at a localhost address (e.g., http://localhost:5173). In Vite's case,
  source files are served as native ES Modules — meaning files are transformed
  on-demand as the browser requests them, rather than bundled upfront.

Simple way:
  It spins up a temporary mini web server on your own computer so you can open
  your React app in a browser at an address like http://localhost:5173.
  This only works on your machine; no one else can see it.

Summary of key commands:
  npm install       — downloads all dependencies into node_modules/
  npm run dev       — starts local dev server, creates .vite cache only
  npm run build     — creates the dist/ folder with final production files
  npm run preview   — serves the dist/ folder locally to test the build

--------------------------------------------------------------------------------
5. THE FLOW — FROM CODE TO BROWSER
--------------------------------------------------------------------------------

1. You write components in .jsx or .tsx files.
2. Vite detects a browser request and transforms your JSX into JavaScript
   on the fly using the React plugin.
3. React runs in the browser. It builds a Virtual DOM from your components.
4. react-dom takes that Virtual DOM and renders it into the real HTML page
   (specifically into the <div id="root"> element in index.html).
5. When state changes, React diffs the old and new Virtual DOM and updates
   only the changed parts of the real page.
